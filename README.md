Projeto Gestor Bancário - CRUD Multi-Entidade📘 Descrição do ProjetoEste projeto foi desenvolvido com fins pedagógicos para alunos do Curso Profissional de Gestão e Programação de Sistemas Informáticos (GPSI) – 10.º ano.O objetivo principal é demonstrar como implementar operações CRUD (Create, Read, Update, Delete) complexas em Python, envolvendo múltiplas entidades relacionadas, utilizando:Funções (sem classes)Dicionários aninhadosSeparação modular por ficheiros (Arquitetura em camadas)Validação de dados rigorosaRelacionamentos entre entidades (IDs estrangeiros)O projeto simula um sistema bancário completo com gestão de Bancários, Clientes, Bancos e Contas.🎯 Objetivos PedagógicosCom este projeto os alunos devem aprender a:Organizar sistemas complexos em múltiplos módulos Python.Gerir relacionamentos entre diferentes dicionários (ex: associar um cliente a um bancário).Implementar lógicas de negócio avançadas (ex: impedir a remoção de um banco se este tiver contas).Validar dados críticos como NIF (9 dígitos), NIB (21 dígitos) e correlação idade/data de nascimento.Trabalhar com fluxos de menus em cascata.📂 Estrutura do ProjetoPlaintext.
└── sistema_bancario
     ├── main.py           # Menu principal e submenus de navegação
     ├── bancario.py       # Lógica e CRUD da entidade Funcionário Bancário
     ├── cliente.py        # Lógica e CRUD da entidade Cliente
     ├── banco.py          # Lógica e CRUD da entidade Instituição Bancária
     ├── conta_bancaria.py # Lógica e CRUD das Contas (corrente/poupança)
     ├── utils.py          # Validadores e geradores de IDs globais
└── README.md
Descrição dos Módulosmain.py: Gere a interface de utilizador. Não processa dados, apenas encaminha as escolhas do utilizador para as funções corretas.bancario.py / cliente.py / banco.py: Módulos que gerem as entidades base. Incluem validações específicas e armazenamento em memória.conta_bancaria.py: Módulo de integração que relaciona Clientes e Bancos para a criação de contas.utils.py: Biblioteca de funções utilitárias para validação de E-mail, NIF, NIB, Datas e cálculos de idade.👤 Estrutura das Entidades PrincipaisEntidadeAtributos PrincipaisExemplo de IDBancárioNome, NIF, Email, Morada, Data Nasc.B001ClienteNome, Idade, NIF, Email, Trabalho, Bancário Resp.C001BancoNome, NIB (21 dígitos), Email, TelefoneBN001ContaTipo (Corrente/Poupança), Saldo, Cliente, BancoCT001▶️ Como Executar o Projeto1️⃣ Garantir que o Python 3.x está instalado.2️⃣ Ter todos os ficheiros (.py) na mesma pasta.3️⃣ Executar no terminal:Bashpython main.py
4️⃣ Seguir as instruções dos menus (Nota: Deve criar um Bancário antes de um Cliente, e um Banco antes de uma Conta).📚 Conceitos TrabalhadosEste projeto permite consolidar:Modularização: Importação de funções entre múltiplos ficheiros.Relacionamento de Dados: Uso de IDs para ligar informações entre dicionários distintos.Tratamento de Exceções: Uso de try/except para validar entradas numéricas.Lógica de Controlo: Verificação de existência de dados antes de permitir operações (ex: consultar ID inexistente).Manipulação de Strings e Datas: Formatação de moedas e validação de formatos ISO (YYYY-MM-DD).👨‍🏫 Utilização em Sala de AulaEste projeto é o passo seguinte ao CRUD simples, sendo ideal para:Demonstrar a importância da organização de código em projetos de média escala.Exercícios de "Integridade Referencial" (o que acontece se apagar um cliente que tem conta?).Avaliação de lógica de programação estruturada.
